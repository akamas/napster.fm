Demandarán por usar esto?

Absolutamente.

No.

Entonces, cómo funciona esto?

En cuanto a la música streaming, esencialmente sólo tomaré los contenidos existentes en YouTube basado en metadatos (títulos, artistas, etc.) de la base de datos Discogs.

¿Así que todo el servicio depende de menores ineficiencias en el sistema de detección de la piratería de YouTube?

Shhh... Probablemente no duele, pero no, no necesariamente; No seas demasiado selectivo sobre la versión de una canción que buscas.

Windows 8 presentó algunas ideas interesantes y un UX casi pasable, pero en su mayor parte creo que el nuevo estilo Metro / tendencia plana parece genial.

Además, me gustó la idea de reimaginación Napster – originalmente un Windows 95 escritorio interfaz – como una aplicación Web moderna 8-ish para Windows.

No tengo nada mejor que hacer con mi vida; ¿Cómo puedo contribuir a Peer.fm?

Como se mencionó en la página de inicio, todo se desarrolla al aire libre en GitHub; cualquier ayuda sería brillante. O algo por correo electrónico antes de enviar cualquier solicitud de tirón, pero por favor no dude en iniciar peticiones ni informes de fallos en el problemas de sistema.

Si decido que debería contribuir, ¿qué detalles técnicos sobre Peer.fm debería tener en cuenta?

1. para gestionar todos nuestros datos, estoy utilizando el almacén de datos "servidor" en tiempo real fantástico Bases avanzadas (YC S11). Como consecuencia, además de un conjunto de reglas de validación de base, esto es pura ingeniería del cliente; grado A JS los hackers solamente.

2. que el SDK de base y la API de YouTube, las principales bibliotecas y estoy usando los marcos son: Cierre de Google, AngularJS, Azúcar y Bootstrap (son muy brillantes).

3. si estás acostumbrado a recorriendo el árbol DOM con jQuery, mi ClosureQuery base de cierre puede parecer un poco familiar.

¿Qué específicamente podrías usar ayuda?

Para empezar, estoy abierto a cualquier propuesta contribución al proyecto. Dicho esto:

1. automatización de pruebas: Peer.fm actualmente tiene cero cobertura de prueba y no tengo ningún plan para escribir cualquier yo mismo en un futuro próximo; Esto podría ser un proyecto a otro para manejar.

2. ilustraciones: actualmente llevo el logo de Akonadi para el Peer.fm favicon (ninguna significación especial; KDE es simplemente impresionante).

3. documentación: no hay ninguno en este momento.
