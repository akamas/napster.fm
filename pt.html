Então, como isso funciona?

Na medida em que o streaming de música, essencialmente só vim pegar o conteúdo existente no YouTube, com base em metadados (títulos, artistas, etc.) do banco de dados Discogs.

Então, todo o serviço é dependente menores ineficiências no sistema de detecção de pirataria do YouTube?

Shhh... Provavelmente não doem, mas não, não necessariamente; Não fique muito seletivo sobre qual versão de uma música que você procura.

Fresco. Então por que é que Peer.fm lembram vagamente Windows 8?

Windows 8 introduziu algumas idéias interessantes e um UX quase passável, mas a maior parte eu só acho que o novo estilo de Metro / plana tendência parece legal.

Além disso, eu gostei da idéia de releitura Napster – originalmente uma interface Windows 95 desktop – como um Windows 8 moderno aplicativo da Web.

Se eu decidir contribuir, que detalhes técnicos sobre Peer.fm deveria saber?

1. para gerenciar todos os nossos dados, estou usando a loja fantástica "bombardeio" de dados em tempo real Firebase (S11 YC). Como conseqüência, além de um conjunto de regras de validação de base, isto é pura engenharia de cliente; hackers de classe A JS somente.

2. além da Firebase SDK e API do YouTube, a grandes bibliotecas e frameworks que estou usando são: Fechamento do Google, AngularJS, Açúcar e Bootstrap (eles são todos brilhantes).

3. se você está acostumado a percorrer o DOM com o jQuery, meu ClosureQuery de encerramento-baseado pode parecer um pouco familiar.
